2. Граничные сценарии (Edge Cases):
- Если платёжка отправит два веб-хука, то нужны 'x-idempotencety-key' иначе могут быть дубли
- Что если очередь переполнена или под консьюмера упал
- Кнопка "Купить в один клик" - проверить на антиспам/даблклики => не должно создастся 2 дублирующие заявки
- У юзера баланс: 100 делает покупку на 101 => ошибка "недостаточно средств"
- У юзера баланс: 100 делает покупку на 99 => успешная покупка/заявка
- У юзера баланс: 100 делает покупку на 100 => успешная покупка/заявка
- 'x-idempotencety-key' для всех сообщений которые проходят через кролика
- Бот падает/крашиться при передачи юзеру пета
- Внешнее АПИ для расчета курса не отвечает, что делаем ? расчитываем по предыдущему курсу ? 
- Наш сервер лежал в то время как веб-хук пытался достучаться до нас
